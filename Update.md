3. Personal Project Retrospective
What Went Well

The most significant achievement during this sprint was successfully creating the Jira activities and learning how to use an emulator and how to get things ready to be shipped as an android app rather than a web app (which I have never done before).

What Did Not Go Well

The main challenges during this sprint were related to development environment configuration. In particular, the Android emulator initially had difficulty connecting to the Metro bundler, which prevented the application from loading correctly. Resolving this is requiring troubleshooting network connections and emulator settings.

These challenges slowed development and they are typical issues encountered during the early stages of mobile development.

What I Learned

This sprint reinforced several important agile and development concepts:

Early project stages often require significant time spent on environment setup.

Troubleshooting and debugging are essential parts of the development process.

Achieving small milestones—such as successfully launching the application—can represent major progress in a project.

Additionally, I gained practical experience with the React Native development process, including how the Metro bundler serves JavaScript code to the mobile application.

Improvements for the Next Sprint

With the development environment now mostly configured, the next sprint will focus on building functional features of the Household Command Center application. Planned work includes:

Implementing the main household dashboard

Adding shared task management features

Developing a family scheduling component

Creating a shared grocery list feature

These features will move the project from a development prototype toward a usable household productivity tool.
